# Claude Code prompt — v2.1 fix round (live QA punch list)

QA'd against https://www.littlefables.ai/read on desktop, including a cleared-storage first-run test. The v2 skeleton is live and the family pack renders, but the build shipped demo data, missing audio, and several PRD violations. Fix in priority order; reference docs: `design/PRD-story-world-v2.md`, `docs/voice-architecture.md`, `design/handoff-v2/`, `content/CONVERSION-NOTES.md`.

## P0 — broken or wrong

1. **First-run flow is missing.** Cleared storage → lands on Home with Bear pre-selected; the buddy carousel never appears. PRD R8: first launch → carousel → pick → Home. Route exists (`/read/buddy`?) but is not the entry gate. Fix: no persisted buddy ⇒ redirect to the carousel before Home.
2. **Demo world-memory leaked to production.** With an empty choiceLog the buddy says "The web bridge you built is still standing!" — the handoff's mock line. The speech line must derive from real `WorldState.choiceLog` (most recent choice summary); with no history it's a greeting ("Good morning, Azad! Pick our first story!"). Grep for every other `data.js` mock string that shipped (greet/memory/meta lines) and replace with real-state derivations.
3. **Miko lost its book and its art.** "Miko and the Wobbly Bridge" appears as a *quick story* with a generic line-art cover. It must be the flagship **3-chapter book** using the real ink-and-wash art that exists in `public/art/`: `miko-cover.jpg` as cover, per-page art in the reader, scene thumbnails on the chapter map. (Chapters: Zoomtown/bridge discovery → belly breath/Tara's web → crossing + night, using the art mapping from the original design handoff `design/handoff/app/story-data.js`.)
4. **No generated audio → no word highlighting.** Zero `/audio/*` assets are deployed; the reader falls back to device synth and no highlight runs. Run `scripts/generate-audio.ts` for pack-000 + Miko + starters (narrator voice), commit/deploy the audio + alignment JSON, and verify highlighting is driven by the timestamps. If the script doesn't exist or wasn't finished, build it now per `docs/voice-architecture.md`.
5. **Chapter-map lock copy is semantically wrong.** Future chapters of *complete* books (e.g., Moose ch. 2–3, fully converted content) show "Not yet… Mom is still painting this one ✦". That line is exclusively the **draft/art-pending** state. Sequential not-yet-unlocked chapters show the closed-book "Not yet…" treatment only. Also: finished chapters must show their scene-art thumbnail + ✓ (currently absent).
6. **Home has no action when nothing is mid-flight.** "Today's adventure — Pick a story from your shelf" is an instruction with no CTA and placeholder art — and it leaves the screen with zero coral. Implement the real Today's adventure: deterministic daily pick (rotate unfinished → unread → least-recently-read), show that story's cover + title + coral ▶, keep the daily-quest line.
7. **Reader doesn't fit the viewport.** Progress bar and controls sit below the fold at 1290×703 (and will on iPad too). Kid surfaces never scroll in the reader: lay out art/text/controls within 100dvh (shrink art panel first, then text card gets internal scroll only if unavoidable).
8. **Pack paging destroyed the wind-down pacing.** Cozy Circle page 1 crams ~6 authored beats (whole opening through "I've got this.") into one wall of text. The originals are written in stanza beats separated by `---`. Update `scripts/convert_family_stories.py`: never merge across the original `---` breaks; add per-story paging config (`maxWords`, `beatPerPage`) with `beatPerPage: true` for The Cozy Circle and Papa Gets the Moon's Rest chapter; regenerate pack-000; sanity-check Moose page counts after (if chapters exceed ~30 pages, split Moose to 4 chapters).
9. **Branding remnant:** document title/tab still "Azad's Story World" → "Little Fables" everywhere user-visible metadata appears.
