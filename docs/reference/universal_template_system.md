# Universal Modular Template System
## For AI Children's Book Illustration Projects

## System Overview

This modular approach separates **consistent elements** (never change) from **customizable elements** (change per scene), enabling efficient, high-quality illustration production for any children's book project.

## Module 4: Assembly Guide & Quality Control

### Step-by-Step Assembly Process

**Step 1: Foundation Setup**
1. Copy Base Style Template (never modify original)
2. Verify style specifications match your book's established aesthetic
3. Confirm technical requirements are current

**Step 2: Character Integration**
1. Select relevant Character Specification modules
2. Prioritize primary character (90% visibility)
3. Add supporting characters with appropriate hierarchy
4. Verify cultural authenticity and age accuracy

**Step 3: Scene Customization**
1. Complete Scene Planning Framework
2. Define activity and emotional context
3. Establish environmental details
4. Set visibility hierarchy for AI comprehension

**Step 4: Prompt Assembly**
```
PROMPT STRUCTURE:
1. Character Priority Statement: "CHARACTER ACCURACY IS HIGHEST PRIORITY"
2. Primary Character Specifications: [Full character module]
3. Supporting Character Specifications: [Additional characters]
4. Style Specifications: [Full style template]
5. Scene Specifications: [Customized scene details]
6. Quality Requirements: [Technical and consistency requirements]
7. Error Prevention: [Negation language and specific avoidances]
```

**Step 5: Quality Verification**
- [ ] Character specifications come first in prompt
- [ ] All distinctive features explicitly stated
- [ ] Style consistency with previous illustrations
- [ ] Age-appropriate content and representation
- [ ] Cultural authenticity maintained
- [ ] Recognition test criteria included

### Troubleshooting Common Issues

**Character Consistency Problems:**
- Solution: Strengthen character priority language
- Solution: Add more specific negation statements
- Solution: Reference successful previous illustrations explicitly

**Style Drift Issues:**
- Solution: Verify Base Style Template hasn't been modified
- Solution: Strengthen style anchoring language
- Solution: Add more specific artistic reference points

**Age Inappropriateness:**
- Solution: Add explicit age-specific developmental language
- Solution: Include safety and appropriateness requirements
- Solution: Strengthen negation language for inappropriate elements

**Technical Quality Issues:**
- Solution: Verify resolution and format requirements
- Solution: Add specific quality control language
- Solution: Include print-readiness specifications
